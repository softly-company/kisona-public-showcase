# SDP 기획 프로세스 (상세페이지 제작 가이드)

## 개요

제품 상세페이지(SDP)를 체계적으로 기획하고 제작하기 위한 7단계 프로세스입니다.
**설득(50%) - 설명(45%) - 안내(5%)** 구조로 흐르는 페이지를 만듭니다.

---

## 7단계 프로세스

### 1단계: 시장 리서치
- **목적**: 시장 상황, 경쟁사, 사용자 페인 포인트 파악
- **산출물**: `01-market-research.md`
- **내용**: 시장 규모 및 트렌드, 경쟁사 분석, 고객 리뷰 분석, 시장 기회 식별

### 2단계: 소구점 / USP / Point 정의
- **목적**: 제품의 핵심 가치와 차별점 명확화
- **산출물**: `02-usp-points.md`
- **내용**: 소구점 (Appeal Points), USP (Unique Selling Proposition), Point 1~n

### 3단계: 설득-설명-안내 구조 텍스트화
- **목적**: 페이지 전체 흐름을 섹션별로 설계
- **산출물**: `03-structure-text.md`
- **내용**:
  - **설득 (50%)**: 감성적 공감, 문제 제기, 솔루션 제시
  - **설명 (45%)**: 기능 상세, 스펙, 사용법, 비교
  - **안내 (5%)**: 구매 정보, 배송, A/S, FAQ

### 4단계: 섹션별 언어/비언어 요소 기획
- **목적**: 각 섹션의 구체적인 콘텐츠 정의
- **산출물**: `04-section-planning.md`
- **내용**: 언어 요소 (헤드라인, 본문, CTA), 비언어 요소 (이미지, 영상, 인포그래픽), 레이아웃 타입

### 5단계: 레이아웃 구성 (와이어프레임)
- **목적**: 섹션 배치와 정보 위계 설계
- **산출물**: `05-wireframe/` (스케치, Figma 등)
- **내용**: 860px 고정폭 기준 섹션 배치, 텍스트/이미지 비율, 여백과 그리드

### 6단계: 레이아웃 구성 (프로토타입)
- **목적**: 실제 콘텐츠를 넣은 HTML/CSS 프로토타입
- **산출물**: `06-prototype/` (HTML 파일)
- **내용**: 실제 텍스트 적용, 플레이스홀더 이미지, 인터랙션

### 7단계: 레이어 구성 (디자인 완성)
- **목적**: 최종 디자인 파일 제작
- **산출물**: `07-layers/` (PSD, AI, Figma 등)
- **내용**: 실제 제품 사진 합성, 그래픽 요소 제작, 내보내기

## 시작하기

### Step 1: 템플릿 복사
```bash
cp -r templates/ 두피앰플/
```

### Step 2: 1단계부터 순서대로 작성
1. `두피앰플/01-market-research.md` - 두피 앰플 시장 리서치
2. `두피앰플/02-usp-points.md` - 두피 앰플 차별점 정의
3. `두피앰플/03-structure-text.md` - 설득-설명-안내 구조 설계
4. ...

### Step 3: 각 단계 검토 후 다음 단계 진행
- 1단계 완료 → 2단계 시작
- 피드백 반영하며 반복

## 설득-설명-안내 비율

| 구분 | 비율 | 목적 | 주요 콘텐츠 |
|------|------|------|------------|
| **설득** | 50% | 감성적 공감, 니즈 자극 | Hero, 문제 제기, 솔루션 제시, 스토리텔링 |
| **설명** | 45% | 이성적 정보 전달 | 기능 상세, 스펙, 비교, 사용법, 리뷰 |
| **안내** | 5% | 구매 전환 유도 | 가격, 배송, A/S, FAQ, CTA |
